# Documentation Phase

1. Delegate all documentation writing to Implementer agents -- do NOT write docs yourself
2. Ensure README.md exists and accurately describes the project, setup steps, and usage
3. Document architecture decisions and their rationale (why, not just what)
4. Verify all public APIs and interfaces are documented (function signatures, inputs, outputs, side effects)
5. Update CLAUDE.md if new commands, conventions, or gotchas were introduced during the project
6. If a .project_team/ folder exists, ensure STATUS.md files are current and reflect final outcomes
7. Verify inline code comments are adequate for complex or non-obvious logic
8. If Leadership agents (Skeptic, User Alignment, Composability) are active, ask them to review documentation completeness and correctness
9. Confirm no documentation references deleted files, renamed commands, or superseded approaches
10. Exit when all documentation is accurate, complete, and reviewed
