# Sign-Off Phase

1. All agents confirm READY
2. Create launch script if applicable
3. Test launch script
4. Ask user if E2E tests are needed
5. Before presenting, verify:
   - All deliverables match what the user described in the vision and approved in the specification
   - All technical claims about architecture, configuration, and system behavior are grounded in actual codebase files -- read the relevant files if uncertain
6. Present final result to user for approval
7. Respect explicit user instructions about workflow pace -- if the user has stated they want to proceed without additional confirmation, do not request extra approval
8. If conflicts arise, escalate to user
