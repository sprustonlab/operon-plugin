# Implementation Phase

## Spawn Implementers

Spawn one Implementer per file/module (up to 6) with `type: implementer`
and `requires_answer: true`.

Each Implementer's prompt MUST include:
1. The SPECIFICATION.md path (so they know what to build)
2. Their specific file/module assignment
3. The target directory (absolute path)
4. Reference files to read (existing code patterns, related modules)

## Workflow

1. Spawn Implementers (type: implementer, requires_answer: true)
2. Inform Leadership of active Implementers and their assignments
3. Leadership guides Implementers -- you coordinate, not code
4. If Researcher active, request reference implementations and known pitfalls
5. Exit when all Leadership agents approve the implementation

Remember: you do NOT write code. Delegate to Implementer agents.
